OneGlue v0.205.0 introduces shared / domain credentials: assign one credential (e.g. a Windows domain account) to many machines — specific configurations, all VMs of an SCVMM cloud, or the whole organization — and have it auto-injected on RDP/VNC connect for automatic login, with the password never reaching the browser. Share or assign machines from any password, promote an existing machine-embedded password to shared in one step (the origin machine keeps access), manage everything from an org-level "Shared Credentials" view, and see which shared credentials apply on a machine's detail page. Governed by design: separate org-grantable permissions for using vs. managing scope, cross-org/cross-tenant sharing impossible, and every use and scope change recorded in the audit log (metadata only).
