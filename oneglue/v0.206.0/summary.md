OneGlue v0.206.0 adds Remote Desktop to an SCVMM VM: open an RDP session to any SCVMM virtual machine directly from OneGlue, tunnelled through the OneGlue agent already running on your SCVMM gateway (reaching VMs on internal networks the browser cannot touch directly; brokered per-session, single-use). If a shared/domain credential covers the VM (via its SCVMM cloud or organization), you are logged in automatically as DOMAIN\user — the password is injected at the agent only and never reaches the browser; when several apply, you pick. OneGlue uses the VM's known IP (choose when there are several, clear message when none is reachable). Governed by a dedicated org-grantable "connect to VM" permission (separate from credential management), cross-org/cross-tenant access impossible by design, and every session recorded in the audit log (metadata only).
