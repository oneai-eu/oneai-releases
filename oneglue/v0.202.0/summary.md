OneGlue v0.202.0 brings end-to-end SCVMM virtual-machine management: edit a VM's identity live (name/owner/notes) and its CPU/memory (with a power-off confirmation), see a per-VM detailed view (disks, network, memory, power state) with last-synced + Refresh, expand and add virtual disks, cap disk throughput with Storage QoS (min/max IOPS or a named policy where the fabric supports it), remove a disk, and delete a VM. Every write action is permission-controlled per action, risk-graduated (harmless changes are one click; disruptive or destructive ones require an explicit type-the-VM-name confirmation), and recorded in the audit log.
