# v1.0.4 — Developer Changelog

**Release Date:** 2026-05-15
**GitHub Release:** [v1.0.4](https://github.com/0codekit/oneai/releases/tag/v1.0.4)

## Refactor

- Jeder Block aus Reasoning- und Tool-Schritten erscheint an der Stelle, an der er tatsächlich passiert ist — eine Antwort mit 'Tool → Text → Tool' wird also als '[Leiste] [Text] [Leiste] [Text]' dargestellt.
- Die automatischen Chat-Titel werden jetzt vom internen Summarization-Modell auf OneStack erzeugt statt über eine externe API. Vorteil: Eure Chat-Inhalte verlassen die EU-Infrastruktur nicht mehr für diese kleine Hilfsaufgabe und werden nicht vom Token-Pool eurer Organisation abgezogen.
- Embedding-Chunk-Vorschauen werden jetzt vom Summarization-Modell statt vom Vision-Modell erzeugt — passender für eine reine Textaufgabe. Toter Code rund um die Vision-Modell-Gateway-Anbindung wurde entfernt.

## Docs

- Interne Dokumentation zum Branch- und Release-Modell sowie schlankere Agent-Regeln — keine sichtbaren Auswirkungen für Nutzer.

## Bugfix

- Wenn die Verbindung während einer langen Antwort kurz abreißt (z. B. durch Proxy-Timeouts), wird der Stream automatisch wiederhergestellt und der bereits generierte Text bleibt erhalten.
- Wenn ein Prompt von einer Compliance-Regel blockiert wird, wird er nicht mehr gelöscht — Admins sehen den ursprünglichen Text samt Anhängen in der Audit-Ansicht und können ihn nach Freigabe sauber wiederherstellen.
- Snippets in der Quellen-Übersicht werden wieder als reiner Text gerendert — der vorherige Markdown-Renderer hat das kompakte Chip-Layout gesprengt.
- Beim Erreichen von Nutzungs-Schwellwerten werden jetzt nur noch Rechnungen ab einem Mindestbetrag erstellt, und doppelte Rechnungen sind durch eine Idempotenzprüfung ausgeschlossen.
- Der Audit-Details-Dialog läuft bei langen blockierten Nachrichten nicht mehr über. Die IBAN-Erkennung trifft deutlich seltener False Positives (z. B. auf Hashes oder Git-Refs), und Admins sehen jetzt direkt den tatsächlich erkannten Text statt nur den Mustertyp.

## Ui

- Während des Streamings zeigt eine zentrale Live-Ansicht die kompletten Schritte mit Auto-Scroll. Nach Abschluss erscheinen zwei Pills — 'Schritte' und 'Quellen' — und das Artefakt-Tray ist jetzt eine volle Karte mit aufklappbarer Historie aller Artefakte des Chats.
- Tool-Aufrufe und Quellen werden jetzt als kompakte Pill-Leiste am Kopf jeder Antwort dargestellt. Favicons für Web-Quellen werden über einen sicheren Backend-Proxy geladen, statt direkt von Google — eure IP-Adresse und euer Surfverhalten landen nicht mehr bei externen Diensten.
- Die Tabs im Subscription-Bereich (Übersicht, Abrechnung, Nutzung, Team-Quoten, Add-ons) sehen auf allen Bildschirmgrößen aufgeräumter aus und passen sich besser an.

## Security

- Die Software-Bill-of-Materials wird jetzt automatisch erstellt — unterstützt zusätzliche Schwachstellen-Scans neben Dependabot. Keine sichtbare Änderung für Nutzer.

## Feature

- Modellpreise werden zur Laufzeit aus dem zentralen oneai-releases-Repo geladen statt fest im Code zu stehen. Im Subscription-Bereich gibt es einen neuen Dialog, der die aktuellen Preise aller verfügbaren Modelle anzeigt.
- Alle in einem Chat erstellten oder genutzten Artefakte sind jetzt in einem festen Tray über dem Eingabefeld auf einen Klick erreichbar. Inline-@artefakt-Verweise öffnen Artefakte direkt im Seitenpanel statt euch wegzunavigieren. Die Distillation-Buttons sind vorerst ausgeblendet — Funktion ist intern erhalten, kommt in überarbeiteter Form zurück.
- Nach einer Web-Recherche der KI erscheinen die genutzten Web-Quellen jetzt direkt unter der Antwort und können im 'Alle Quellen'-Panel zusammen mit Datei-Quellen eingesehen werden. Favicons der zitierten Seiten werden über euren Server geladen — nicht mehr über Google.

## Performance

- Dokumenten-Zusammenfassungen und Embeddings laufen jetzt parallel zum Vision-Modell statt nacheinander — Verarbeitung ist spürbar schneller. Ein Bug ist behoben, durch den PDF-Seiten doppelt indexiert werden konnten (einmal Rohtext, einmal OCR).
- Ungenutzte und mit Primärschlüsseln redundante Datenbankindizes wurden entfernt — leicht schnellere Schreibvorgänge, keine sichtbaren Änderungen für Nutzer.
- Die Vektordatenbank für die semantische Dokumentensuche hält Indizes jetzt auf der Festplatte statt im RAM und nutzt komprimierte Vektoren mit Rescore-Nachbearbeitung. Massiv weniger Speicherverbrauch bei unverändert hoher Suchqualität.

## Deps

- Routine-Update von Drittanbieter-Bibliotheken durch Dependabot — keine sichtbaren Änderungen, schließt aber bekannte Sicherheitslücken.
