# v0.3.4-beta — Developer Changelog

**Release Date:** 2026-03-18
**GitHub Release:** [v0.3.4-beta](https://github.com/0codekit/oneai/releases/tag/v0.3.4-beta)

## Breaking Changes

- **Persona system migrated from project-level (many-to-many) to chat-level (single persona per chat). The project_personas junction table is dropped. Existing position=0 persona becomes the project default.**
- **Space files list API response structure changed from flat array to object with files array, totalFiles count, and optional hasNextPage flag.**

## Feature

- Bilder können jetzt auch mit Google Imagen 4 generiert werden — in drei Geschwindigkeits- und Qualitätsstufen.
- GitHub-Spaces erstellen automatisch Übersichtsdateien und reichern Code-Dateien mit Kontext an — für präzisere KI-Antworten.
- Artefakte können geteilt, per Link geöffnet, durchsucht und nach verschiedenen Kriterien sortiert werden.
- Personas sind jetzt pro Chat wählbar — mehr Flexibilität bei verschiedenen Gesprächen im selben Projekt.
- Projekte zeigen jetzt Tabs für Chats, Dateien und Artefakte. Chat-Listen und Projekt-Übersichten wurden überarbeitet.
- Jede KI-Antwort zeigt jetzt an, welches Modell sie generiert hat.
- Während die KI zwischen mehreren Tool-Aufrufen nachdenkt, wird jetzt ein Denk-Indikator angezeigt statt eines leeren Bildschirms.

## Bugfix

- Outlook-E-Mails werden jetzt inkrementell synchronisiert und abgelaufene Webhook-Abonnements automatisch erneuert.
- Das Erstellen von Artefakten wurde flüssiger — der richtige Ladeindikator wird jetzt bei der richtigen Aktion angezeigt.
- Lange KI-Denkphasen bei Anthropic-Modellen brechen nicht mehr wegen Zeitüberschreitung ab.
- Dropdown-Menüs und Auswahl-Felder funktionieren jetzt nach der UI-Migration wieder korrekt.
- Externe Integrationen erhalten jetzt auch die Organisations-ID in der Verbindungsantwort.

## Refactor

- Die gesamte UI-Komponentenbibliothek wurde von Radix UI auf Base UI umgestellt — schnellere Ladezeiten und modernere Grundlage.
- Internes Load Balancing wird jetzt vom HAProxy-Sidecar übernommen statt im Anwendungscode.
- Code-Dateitypen-Liste in ein gemeinsames Modul verschoben, um Duplikate zu vermeiden.

## Docs

- Interne Dokumentation der geplanten Anthropic Contextual Retrieval Pipeline.
