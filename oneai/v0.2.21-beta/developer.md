# v0.2.21-beta — Developer Changelog

**Release Date:** 2026-02-24
**GitHub Release:** [v0.2.21-beta](https://github.com/0codekit/oneai/releases/tag/v0.2.21-beta)

## Breaking Changes

- **Per-user preferred_model column dropped from users table. Model selection now managed at org level (org_base_models table + default_model_id) and team level (team_models table). Users' previously saved model preferences are lost — org default model applies instead.**
- **New required environment variables: VISION_URL, VISION_KEY, VISION_MODEL. Tesseract.js, @napi-rs/canvas, and Google Gemini (@google/genai) no longer used for image analysis and chunk summarization in embedding pipeline.**
- **Admin routes restructured: /dashboard/payment → /dashboard/admin/payment, /dashboard/private-models → /dashboard/admin/private-models, /dashboard/scim → /dashboard/admin/scim. New index at /dashboard/admin for org model settings.**

## Feature

- Admins können zentral steuern, welche KI-Modelle verfügbar sind, ein Standard-Modell festlegen und Teams gezielt zusätzliche Modelle zuweisen.
- Bilder und gescannte Dokumente werden jetzt von einem selbst gehosteten Vision-KI-Modell verarbeitet statt von klassischer Texterkennung und externen APIs.

## Bugfix

- Chat-Karten zeigen ein aufgeräumteres Design, Artefakte können sicher gelöscht werden, die Artefakt-Ansicht ist wieder scrollbar und der Datei-Ersetzen-Button funktioniert wieder.

## Performance

- Bei Spaces mit vielen Dateien konnte der Browser abstürzen, weil zu viele Daten geladen wurden. Ein neuer schlanker Status-Endpunkt reduziert den Datenverkehr um über 99%.
- Die PDF-Pipeline wurde grundlegend überarbeitet: einfachere Logik, zuverlässigere Ergebnisse bei gescannten Dokumenten und PDFs mit Bildern.

## Deps

- Regelmäßiges Update aller externen Bibliotheken für Sicherheit und Stabilität.
- Backend-Framework Yedra auf neueste Version aktualisiert mit Biome-Konfigurationsanpassungen.
